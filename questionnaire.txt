##Les conditions
L'évaluation entre parenthèses doit retourner quelle valeur pour que l'algorithme dans le bloc du if soit executé ?
Une variable flag est initialisée a false, si j'ecris if(!flag){}else{} :
- le code entre les accolades du if sera executé
- le code entre les accolades du else sera executé
- aucun des deux blocs ne sera executé
- le code est incorrect
Dans un bloc de conditions multiples (if else if), si le test valide les deux conditions :
- seul le else if est executé
- les deux blocs sont executés
- le bloc if est executé
- si, sinon si c'est assez clair que seul le premier est executé
##Les boucles
Une boucle for est plus pratique pour parcourir un tableau, pourquoi ?
Un compteur est décrémenté a chaque itérations de 1 (selectionnez les commentaires justes):

- for(int compteur = 10;compteur>1;compteur--){} // le compteur s'arrete à deux
- while(true){} // ceci est une boucle infinie
- for(...){} // la condition d'une boucle for doit retourner faux pour continuer a boucler
